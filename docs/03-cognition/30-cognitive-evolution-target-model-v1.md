# Cognitive Evolution Target Model V1

## Status

Proposed

---

# Purpose

The Cognitive Evolution Target Model defines how PHYTCOI identifies the next cognitive evolution priority after a Cognitive Audit.

The model does not evolve the organism.

The model does not create new organs.

The model does not modify cognition.

The model only answers:

> What should evolve next?

---

# Relationship With Audit

The Cognitive Audit determines:

* anatomy
* maturity
* integrity

The Evolution Target Model consumes these outputs.

Audit answers:

```text
What exists?
```

Evolution Target answers:

```text
What should improve next?
```

---

# Core Principle

PHYTCOI evolves through bottleneck reduction.

The organism must not evolve randomly.

The organism must not evolve because a new idea exists.

The organism evolves only when a measurable bottleneck exists.

# Evolution Categories

---

## Missing Organ Evolution

Condition:

```text
required organ absent
```

Example:

```text
context missing
```

Priority:

Very High

---

## Immature Organ Evolution

Condition:

```text
organ exists
organ partially assimilated
```

Example:

```text
experience immature
```

Priority:

High

---

## Disconnected Organ Evolution

Condition:

```text
organ exists
cognitive flow broken
```

Example:

```text
knowledge not consumed
```

Priority:

High

---

## Integrity Evolution

Condition:

```text
integrity degraded
```

Example:

```text
broken dependency chain
```

Priority:

Critical

---

## Maturity Evolution

Condition:

```text
organism stable
integrity healthy
next maturity stage blocked
```

Priority:

Medium

---

## Visibility Evolution

Condition:

```text
cognition exists
not visible
```

Example:

```text
not represented in observatory
```

Priority:

Low

---

# Priority Order

Canonical priority order:

```text
1. Integrity

2. Missing Organ

3. Disconnected Organ

4. Immature Organ

5. Maturity Expansion

6. Visibility Improvements
```

The organism always resolves structural problems before expanding cognition.

---

# Blocked Evolution

Evolution is blocked when:

```text
integrity not healthy

cognitive flow broken

required dependencies absent
```

Examples:

```text
planner blocked

executor blocked

governance blocked
```

until cognition becomes sufficiently coherent.

---

# Relationship With Planner

The Evolution Target Model is not a planner.

It does not generate plans.

It does not schedule work.

It only identifies the next recommended cognitive bottleneck.

The Cognitive Evolution Target Model ensures that PHYTCOI evolves through measurable cognitive necessity rather than arbitrary feature growth.

* what is blocking cognition
* what should evolve next
