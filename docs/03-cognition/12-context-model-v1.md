# Context Model v1

## Purpose

The Context organ derives meaning from relationships observed within larger systems.

Context does not create observations.

Context does not create knowledge.

Context does not create relationships.

Its purpose is to understand how the meaning of a relationship changes when additional surrounding conditions are considered.

Context represents the first form of situational cognition.

---

# Core Question

Each cognitive layer answers a different question.

Knowledge answers:

"What is repeatedly true?"

Understanding answers:

"What does it mean?"

Relationship answers:

"How are truths connected?"

Context answers:

"What changes when surrounding conditions are considered?"

---

# Definition

Context is the interpretation of a relationship inside a broader environment.

A relationship alone is insufficient.

The surrounding conditions determine its significance.

Example:

Relationship:

Resolver

↓

depends_on

↓

Authority

This describes a dependency.

Context:

Authority unavailable

Result:

Resolver cannot guarantee truth.

The relationship remains the same.

The meaning changes because the context changed.

---

# Universal Principle

Context must remain universal.

The following is forbidden:

PiHole

↓

depends_on

↓

Windows DNS

The following is valid:

Recursive Resolution Provider

↓

depends_on

↓

Authoritative Name Provider

Context operates on functions, not implementations.

---

# Context Components

Every context contains:

Relationship

Condition

Resulting Meaning

Example:

Relationship:

A depends_on B

Condition:

B unavailable

Meaning:

A capability becomes degraded

low

medium

# Forbidden Outcomes

Context may never:

make decisions

execute actions

modify habitat

change lifecycle

approve actions

Context remains an OBSERVE organ.

# Long-Term Goal

The purpose of Context is not to understand products.

The purpose of Context is to understand systems.

A mature PHYTCOI organism must reason about functions, capabilities and dependencies regardless of vendor, hostname or implementation.
