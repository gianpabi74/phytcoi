# Purpose

This document defines Learning inside the Executor of PHYTCOI.

Learning is the capability through which operational experience influences future execution behavior.

The Executor performs actions.

The Executor observes outcomes.

The Executor accumulates memories.

The Executor extracts experience.

Learning transforms experience into improved future behavior.

Without learning, execution remains repetitive.

With learning, execution becomes adaptive.

---

# Why Learning Exists

An organism that repeatedly performs the same action while ignoring outcomes cannot mature.

Without learning:

the same mistakes repeat

the same hazards reappear

the same recovery paths are rediscovered

the same inefficiencies persist

Learning exists to reduce repeated operational failure and improve future execution quality.

---

# Learning Philosophy

Learning is not memory.

Learning is not knowledge storage.

Learning is behavioral improvement derived from experience.

Memory preserves events.

Experience identifies patterns.

Learning modifies future behavior.

These concepts must remain distinct.
