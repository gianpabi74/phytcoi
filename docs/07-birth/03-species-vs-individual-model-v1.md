# Species vs Individual Model v1

## Status

Approved

## Scope

Defines the distinction between a PHYTCOI species and a PHYTCOI individual.

This distinction is fundamental for portability, seed installation, habitat migration and long-term evolution.

---

# Core Principle

A species is not an individual.

An individual is not a species.

The species defines what a PHYTCOI organism is.

The individual represents a specific living instance of the species.

---

# Species

Species:

PHYTCOI

The species contains the universal biological design.

The species must be portable across habitats.

The species must not contain habitat-specific knowledge.

The species must not contain individual memories.

---

# Species Components

The species includes:

* Constitution
* Anatomy
* Organ Definitions
* Runtime Models
* Cognitive Models
* Governance Models
* Seed Installer
* Observatory Architecture
* DNA Structure
* Germination Logic
* Phenotype Rules

The species defines capabilities.

The species does not define lived experiences.

---

# Individual

An individual is a living organism instantiated from the species.

Example:

PHYT-001

Nest:

NEST-ALPHA

Lifecycle:

OBSERVE

Status:

ALIVE

---

# Individual Components

An individual contains:

* Observations
* Evidence
* Memories
* Experiences
* Knowledge
* Understanding
* Runtime State
* Life Events
* Journals
* Records

These elements are created during life.

They are not part of the species.

---

# Species Repository

The GitHub repository represents the species.

The repository must contain:

* Documentation
* Organ Templates
* Runtime Definitions
* Seed Components
* Systemd Units
* Phenotype Assets
* Constitutional Rules

The repository must not contain:

* Runtime Journals
* Runtime Records
* Temporary State
* Habitat-Specific History

---

# Individual Runtime

The runtime environment contains the living individual.

Example:

/opt/phytcoi

contains the active organism.

The runtime environment is expected to evolve continuously.

The repository remains the reference genome.

---

# Germination

During germination the species becomes an individual.

Species

↓

Seed

↓

Nest

↓

Individual

The resulting organism begins life with no experiences.

Experiences are acquired after birth.

---

# Habitat Independence

The species must remain habitat-independent.

A future seed may be planted in:

* Proxmox
* Hyper-V
* VMware
* Cloud Infrastructure
* Physical Hardware
* Unknown Future Habitats

The species must remain valid regardless of habitat.

---

# Local Names

Local identifiers may exist only during germination and evidence acquisition.

Examples:

* hostnames
* IP addresses
* vendor names
* product names

These are implementation details.

Long-term cognition must operate on:

* functions
* capabilities
* relationships
* evidence

---

# Cognitive Continuity

An individual may develop unique experiences.

These experiences belong to the individual.

The species may evolve through documentation and constitutional changes.

The individual evolves through cognition.

---

# Repository Rule

GitHub represents the species.

Runtime represents the individual.

Species and individual must never be confused.
