# PHYTCOI Cambium Runtime v1

## Purpose

This document defines how growth is designed within a PHYTCOI individual.

The Cambium Runtime is responsible for identifying growth opportunities and producing growth blueprints.

Cambium does not directly modify the individual.

Cambium designs growth.

Other systems realize growth.

---

# Fundamental Principle

Cambium does not execute growth.

Cambium plans growth.

Growth must remain:

* deliberate
* explainable
* reversible
* evidence-based
* governed

Cambium proposes.

Governance authorizes.

Adaptation realizes.

The Cambium Runtime answers:

How should the individual evolve?

# Cambium Architecture

The Cambium Runtime consists of four layers.

---

# Layer 1 — Opportunity Detection

## Purpose

Identify potential improvements.

Questions:

What is missing?

What repeatedly causes difficulty?

What capability would improve survival?

What capability would improve understanding?

Opportunity detection is evidence-driven.

---

# Opportunity Registry

Contains:

* opportunity identifier
* opportunity description
* supporting evidence
* confidence
* expected benefit

---

# Layer 2 — Capability Gap Analysis

## Purpose

Determine capability deficiencies.

Examples:

Missing observation capability.

Missing recovery capability.

Missing classification capability.

Missing governance capability.

Capability gaps identify growth targets.

---

# Gap Registry

Contains:

* gap identifier
* affected organ
* impact assessment
* supporting evidence
* confidence

---

# Layer 3 — Blueprint Generation

## Purpose

Design growth.

A blueprint describes:

* objective
* affected organ
* required capability
* implementation strategy
* rollback strategy
* validation requirements

Blueprints are implementation-independent.

---

# Blueprint Registry

Contains:

* blueprint identifier
* growth objective
* dependencies
* validation criteria
* rollback criteria

Blueprints are authoritative growth proposals.

---

# Layer 4 — Adaptation Proposal Layer

## Purpose

Prepare growth for governance review.

Outputs:

* adaptation proposals
* capability proposals
* extension proposals

This layer does not activate growth.

It proposes growth.

---

# Adaptation Proposal Registry

Contains:

* proposal identifier
* blueprint reference
* confidence
* governance requirements
* expected outcomes
