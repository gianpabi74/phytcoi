# PHYTCOI DNA Runtime v1

## Purpose

This document defines how DNA physically exists within a PHYTCOI individual.

The DNA Runtime is responsible for storing validated habitat truth.

DNA is the authoritative source of knowledge about the habitat currently understood by an individual.

The DNA Runtime exists to ensure that habitat truth remains:

* evidence-based
* traceable
* recoverable
* explainable
* authoritative

---

# Fundamental Principle

DNA is Habitat Truth.

DNA is not:

* opinion
* interpretation
* recommendation
* experience
* prediction
* policy

DNA contains only validated observations about reality.

---

# Core Question

The DNA Runtime answers:

What currently exists in the habitat?

---

# Runtime Ownership

Biological Owner:

DNA System

Runtime Owner:

DNA Service

Primary Consumers:

* World Model
* Governance
* Risk
* Cambium
* Decision Systems

DNA serves the entire organism.

---

# DNA Runtime Objectives

The DNA Runtime must provide:

* habitat truth
* object persistence
* evidence traceability
* confidence management
* classification persistence
* capability persistence

The DNA Runtime is not responsible for reasoning.

---

# One Truth Principle

For every discovered object:

There must be exactly one authoritative DNA record.

Multiple competing truths are prohibited.

The DNA Registry is the authoritative source.

---

# DNA Registry

The DNA Registry is the central storage structure of the DNA Runtime.

The registry contains:

* object records
* capability records
* classification records
* confidence records
* evidence bindings

The registry is authoritative.

---

# DNA Object Model

The DNA Runtime is organized around Objects.

The species does not assume infrastructure-specific assets.

Everything observed is represented as an Object.

Examples:

* server
* service
* device
* process
* API
* storage system
* future unknown object

Object is the universal abstraction.

---

# DNA Object Record

Every object record contains:

Object Identifier

Object Type

Observed Properties

Validated Capabilities

Classification

Confidence

Evidence References

Lifecycle State

Timestamp Metadata

A record without evidence is invalid.

# DNA Capability Records

Capabilities are stored separately from objects.

Examples:

* name_resolution
* identity_management
* recovery
* media_delivery
* object_discovery

Capabilities describe function.

Objects provide capabilities.

---

# DNA Classification Records

Classifications describe what an object appears to be.

Examples:

* Resolution Provider
* Identity Provider
* Recovery Provider
* Hosting Provider

Classifications emerge from evidence.

Classifications are never assumed.

---

# DNA Confidence Model

Every DNA element contains confidence.

Confidence represents belief strength.

Confidence may increase through:

* repeated validation
* independent evidence
* historical consistency

Confidence may decrease through:

* contradictory evidence
* missing observations
* stale information

Confidence is mandatory.

---

# DNA Evidence Binding

Every DNA statement must reference evidence.

DNA may never contain unsupported claims.

Every record must answer:

Why is this believed to be true?

Evidence provides the answer.

---

# DNA Validation

Information enters DNA only after validation.

The lifecycle is:

Observation

↓

Evidence

↓

Hypothesis

↓

Validation

↓

DNA

No direct insertion is permitted.

---

# DNA Lifecycle

DNA records evolve through stages.

Observed

↓

Candidate

↓

Validated

↓

Active

↓

Stale

↓

Archived

The lifecycle preserves history.

---

# DNA Persistence

DNA must survive:

* service restart
* runtime restart
* habitat reboot
* individual restart

DNA persistence is mandatory.

Loss of DNA represents memory loss.

---

# DNA History

DNA is versioned.

Every significant change must preserve:

* previous state
* timestamp
* evidence chain

Historical truth must remain accessible.

---

# DNA Authority Rules

DNA is authoritative for:

* object existence
* object capabilities
* object classifications
* object confidence

DNA is not authoritative for:

* decisions
* recommendations
* governance
* experience
* strategy

Authority boundaries must remain clear.

---

# DNA and World Model

DNA answers:

What exists?

World Model answers:

How reality behaves?

DNA provides facts.

World Model provides understanding.

These responsibilities must remain separate.

---

# DNA and Experience

DNA contains truth.

Experience contains lessons.

A lesson is not a fact.

A fact is not a lesson.

The two systems must remain independent.

---

# DNA and Governance

Governance consumes DNA.

Governance does not own DNA.

Governance may evaluate truth.

Governance may not rewrite truth without evidence.

---

# DNA and Cambium

Cambium consumes DNA.

Growth opportunities emerge from DNA.

Cambium may not modify DNA directly.

Growth must remain evidence-driven.

---

# DNA and Decision Systems

Decision systems consume DNA.

DNA does not contain decisions.

Decision systems may reason using DNA.

Reasoning must remain external.

---

# DNA Storage Domains

The DNA Runtime contains:

Evidence Reference Registry

Together they form the DNA Registry.

---

# DNA Recoverability

DNA must support:

* backup
* restoration
* replay
* reconstruction

Recoverability is mandatory.

---

# DNA Portability

DNA is habitat-specific.

DNA must not be embedded in Seeds.

Seeds contain species truth.

DNA contains habitat truth.

A new individual must build new DNA.

---

# Anti-Patterns

The following are prohibited.

---

## Knowledge Duplication

Multiple truth stores describing the same object.

---

## Evidence-Free Records

Records without evidence references.

---

## Decision Storage

Storing recommendations inside DNA.

---

## Experience Storage

Storing lessons inside DNA.

---

## World Model Storage

Storing causal understanding inside DNA.

---

## Vendor-Centric Records

Assuming product names as primary classification.

Capabilities are primary.

Products are secondary.

---

# Runtime Success Criteria

The DNA Runtime is successful when it can:

* store habitat truth
* preserve evidence
* support validation
* provide authoritative object knowledge
* survive restarts
* support future reasoning systems

while remaining evidence-based.

---

# Final Rule

DNA is not knowledge because it is stored.

DNA is knowledge because it has been validated.

The purpose of the DNA Runtime is to preserve validated habitat truth.

The DNA Runtime is the memory of what exists.

Not why it exists.

Not what should happen next.

Only what has been demonstrated to be true.
